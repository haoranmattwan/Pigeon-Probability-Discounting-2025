# Contributing

Contributions that improve reproducibility, documentation, testing, or accessibility are welcome.

1. Open an issue describing the proposed change and its scientific or technical rationale.
2. Keep the published R specification distinct from extensions or sensitivity analyses.
3. Do not commit research data, model objects, rendered reports, private documents, or copyrighted article PDFs.
4. Add or update validation checks when changing data processing or model equations.
5. Record package versions, random seeds, convergence diagnostics, and deviations from the article.
6. Run the relevant analysis in a clean environment before submitting a pull request.

By contributing code, you agree that it may be distributed under the repository's MIT License. The study data remain governed by the terms stated on OSF.
